Claude 說「好」，但我看不懂它給我的東西

那一大坨字

那天，我鼓起勇氣問了 Claude 怎麼做一個可以新增衣服的頁面。它回答得超快，整整三百行程式碼，密密麻麻排在螢幕上。那些英文字母、括號、分號交錯在一起，像某種外星文。我的第一反應不是「太好了！」，而是默默在心裡問：「所以……我現在要怎麼辦？」

複製、貼上，然後一片空白

我以為自己懂「複製貼上」。把程式碼全選、複製，打開某個檔案，貼進去，存檔，重新整理——不就完成了嗎？沒有。螢幕一片空白，什麼都沒出現。我又試了一次，還是空白。我那時完全不知道這段程式碼應該放在哪個檔案裡，甚至連「哪個檔案」是什麼意思都搞不太清楚。

看不懂也先試試看

那個晚上，我不斷問 Claude：「這段放哪裡？」「這個括號是什麼意思？」「為什麼沒有畫面？」我沒有打算把每一行都搞懂，但我開始試著理解「大概的樣子」——原來這段在說「按下這個按鈕」，原來這行在告訴畫面要顯示什麼。那些模糊的輪廓，一點一點浮現了。「就算看不懂也先試試看」這件事，在那個晚上開始變成我的習慣。

凌晨一點，一個空白的框框

折騰了好幾個小時後，手機螢幕上終於出現了一個可以輸入文字的框框。就只是一個空的框框，什麼功能都還沒有。但我截了圖，傳給 Kyo，她回了一個「耶！」。當時那個感覺，已經夠了。
