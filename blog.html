Dev Notes

開發隨筆

工作室的幕後日誌。做 APP 的過程裡，我們想到了什麼、踩過了什麼坑，以及那些說不定有人也想看的零散思考。

Kyo 的文章

  • （佔位文字）這是第一篇開發隨筆的佔位內文。Kyo 可以在這裡分享設計決策、產品思考，或是某個功能背後的故事。

    （佔位文字）例如：我們最初想解決什麼問題？第一版和現在的版本差了多少？哪些功能是被砍掉又被加回來的？這些都是很值得記錄的事。

  • （佔位文字）第二篇的佔位內文。Kyo 可以在這裡分享設計過程中的取捨、靈感來源，或者使用者測試的觀察與心得。

    （佔位文字）關於 UI 設計的思考、字型的選擇、配色的邏輯……這些細節放在隨筆裡，讀起來很有味道。

Mimi 的文章

  • （佔位文字）Mimi 的第一篇技術隨筆佔位文字。可以分享技術選型、開發踩坑、某個 bug 怎麼找到又怎麼修掉，或是對某個 library 的使用心得。

    （佔位文字）對其他開發者來說，這種誠實的「踩坑記錄」往往比教學文章更有參考價值。寫下來，說不定能幫助到某個正在撞牆的人。

  • （佔位文字）第二篇的佔位內文。Mimi 可以在這裡分享如何在忙碌的日常中找到時間寫程式、如何維持對 side project 的熱情，或是工作室兩人如何協作。

    （佔位文字）不一定要是技術內容，生活與開發的平衡、動力低谷時怎麼撐過去——這些軟性的分享，有時反而是最多人想看的。
